博客草创及更新

本来是想自己写博客网页的，结果发现折腾起来太麻烦了，css和js看得我脑袋晕~终于不再想自己造轮子了（emmm……其实只是懒蛤蛤蛤蛤），作为一个法学学生我还是省点力气吧。

其实github和markdown我之前在用，所以算是轻车熟路，安装一下node.js和hexo就行。于是就用hexo搭了一个博客，以next作为主题。

感谢互联网时代和各位大佬，网上的教程一搜一大把蛤蛤蛤，哝，我就参考了以下一些文章，顺便记录一下这个博客的功能更新，诸位要是有兴趣也可以自己搭一个博客玩玩：

    以下这个知乎回答推荐给小白看：

小修小补

  • 在移动设备下启用NexT主题的目录页面和回到顶部按钮

    https://leaferx.online/2017/02/05/EnableTOConMobile//

  • 代码块自定义样式、修改文章内链接文本样式、修改文章底部#号+标签为图片+标签

    http://blog.ynxiu.com/2016/hexo-next-theme-optimize.html/

  • 修改作者头像变为圆形并旋转

    http://blog.csdn.net/qq_33699981/article/details/72716951/

  • 发布显示更多

    在你觉得适合的位置插入

    <!– more –>

    参考 http://code.wileam.com/build-a-hexo-blog-and-optimize/

  • 开启Scroll Percent、Save Scrol
    在主题配置文件中修改
    scrollpercent: true
    save_scroll: true

  • 动态背景特效
    在主题配置文件中修改如下几个参数为true/false：
    canvas_nest: true
    canvas_lines: true
    three_waves:
    canvas_sphere:

    参考 http://theme-next.iissnan.com/theme-settings.html#use-bg-animation/

  • 添加文章更新时间
    http://blog.csdn.net/ganzhilin520/article/details/79053399/

  • 修改网页底部、文章顶部显示更新时间、统计功能
    1. 主题配置文件,搜索关键字 copyright
    2. 打开主题配置文件 _config.yml ,搜索关键字 updated_at 设置为 true
      • 主题根目录下 npm i –save hexo-wordcount
      • 在主题配置文件中修改post_wordcount
      • item_text: true
      • wordcount: true
      • min2read: true
      • totalcount: true
      • separated_meta: true
详见 <http://www.dragonstyle.win/2017/11/07/Hexo-Next%E4%B8%BB%E9%A2%98%E4%BC%98%E5%8C%96//>

  • 在网站底部加上卜算子访问量
    http://blog.csdn.net/qq_33699981/article/details/72716951
    我被这个命令给坑了：hexo d -g
    具体我也不知道发生了什么，长长的报错之后，git终端对于node和npm都反馈bash:command not found，git还能用，然后windows命令行完全正常。。。
